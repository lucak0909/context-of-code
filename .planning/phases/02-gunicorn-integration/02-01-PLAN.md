<task type="auto">
  <name>Task 2: Open port 5000 in UFW</name>
  <files></files>
  <action>
On the VM (SSH session from Task 1 still open, or re-SSH), check UFW status first:
```
sudo ufw status
```

Two cases:

**Case A — UFW is active:**
Add a rule for port 5000:
```
sudo ufw allow 5000/tcp
sudo ufw status
```
Confirm the output includes a line with `5000/tcp` and `ALLOW`.

**Case B — UFW is inactive:**
No rule is needed. Port 5000 is already accessible. Document this in the task output (note: UFW inactive, no rule required). Do not enable UFW — enabling it would block all ports including SSH (port 2214).

No files are modified on disk for this task — it is a pure firewall configuration step. Record the UFW status (active or inactive) in the Plan 02 SUMMARY so Phase 4 knows the firewall state.
  </action>
  <verify>
    <automated>sudo ufw status (run on VM — either shows `5000/tcp ALLOW` or `Status: inactive`)</automated>
    <manual>If UFW was active: output of `sudo ufw status` must include `5000/tcp` in the ALLOW rules. If UFW was inactive: no action needed, document the state.</manual>
    <sampling_rate>run after this task, before Plan 02 begins</sampling_rate>
  </verify>
  <done>

  </done>
